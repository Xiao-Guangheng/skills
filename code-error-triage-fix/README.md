# 代码缺陷修复助手

> 面向「代码智能辅助与质量保障」赛道的参赛 Skill

一个系统化的代码缺陷诊断与质量保障 Skill，覆盖"问题分诊→缺陷定位→修复验证→单测生成→安全扫描→审查清单"六阶段质量闭环。

---

## 核心能力

### 1. 报错分诊与缺陷定位
- 将报错分为语法/缩进、运行时异常、逻辑缺陷、安全风险四类
- 对每类列出所有可能错因并按优先级排序
- 每条判断绑定具体行号、变量名、调用链等证据

### 2. 交互式追问排查
- 信息不足时用选择题方式追问，每轮不超过 3 个问题
- 每个问题必须能排除至少一类错因
- 逐轮缩小范围，直到能给出明确修复路径

### 3. 单元测试生成
- 自动生成覆盖正常路径、边界条件、异常路径的测试用例
- 适配用户项目已有的测试框架
- 参数不足时先追问再生成

### 4. 代码审查检查清单
- 按安全、逻辑、风格、性能、测试覆盖五个维度组织
- 问题按阻塞、严重、建议三级标记
- 每个建议绑定具体代码行

### 5. 安全风险扫描
- 检查注入、命令执行、路径穿越、弱加密、硬编码密钥、反序列化、XSS/CSRF 等风险
- 默认保守策略：不能确认安全时不放行

## 适合什么时候用

| 场景 | 说明 |
| --- | --- |
| 日常写代码报错 | Traceback 看不懂、缩进报错、类型错误 |
| 单测失败 | 不知道失败原因、不会写测试用例 |
| PR/Diff 审查 | 需要结构化审查清单和风险点 |
| 安全自查 | 提交前检查注入、密钥、权限问题 |
| 代码复盘 | 分析历史缺陷模式、沉淀排查经验 |

## 评分维度覆盖

| 评审维度 | 分值 | 本 Skill 如何覆盖 |
| --- | --- | --- |
| 运行稳定性与鲁棒性 | 30 分 | 异常输入不崩溃，信息不足时追问而非硬猜，安全处理顺序明确 |

## 使用建议

1. 先提供代码片段和完整报错，让 Skill 做初步分诊
2. 如果没有完整 Traceback 或运行环境信息，先补上再做定位
3. 对修复后的代码运行验证步骤，确认问题已解决
4. 安全扫描结果需要人工复核，不要直接当作最终结论

⚠️ **AI 生成内容提醒**：本技能包的所有诊断和建议均由 AI 生成，请勿直接合并到生产分支或作为唯一安全审查依据。
