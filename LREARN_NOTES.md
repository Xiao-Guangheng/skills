# Agent Skills 深度学习笔记：从提示词进阶到可复用能力包


## 一、重新认识 Agent Skills

### 1. 什么是 Agent Skills？
智能体技能（Agent Skills）是一种**模块化的能力扩展包**。每个 Skill 就像一个独立的文件夹，封装了指令、元数据和辅助资源（如脚本、模板）。当用户的真实场景与匹配规则契合时，AI 会**自动挂载**并调用这些技能，瞬间化身该领域的专家。

### 2. Skill ≠ 提示词 (Prompt)
这是最核心的洞见：**提示词是教 AI“做这件事”，而 Skill 是教 AI“遇到这类问题该怎么思考，用什么工具”。**

| 维度 | 提示词（Prompt） | Agent Skill |
| :--- | :--- | :--- |
| **形态** | 一段纯文本 | 一个文件夹（含指令、脚本、资源） |
| **复用性** | 每次都需要复制粘贴 | 一次构建，随处部署（Build once, deploy everywhere） |
| **触发方式** | 用户手动输入 | AI 根据 Metadata（描述）自主路由触发 |
| **可测试性** | 依赖肉眼和体感判断 | 自带自动化验证脚本 |

## 二、核心痛点与“三层架构”解药

### 1. 传统 Prompt 的“Context 陷阱”
如果我们把所有规范、背景、要求全塞进 System Prompt，会导致：
*   **注意力分散**：无关信息干扰推理，幻觉率激增。
*   **成本灾难**：上下文窗口利用率不足 40%，Token 消耗暴涨。

### 2. 破局之道：渐进式披露与三层架构
Agent Skills 的哲学是：**只在需要的时候，加载需要的知识。**

*   **第 1 层：元数据层（Metadata Layer）** 
    *   **常驻内存**：仅包含技能名称（name）和描述（description）。
    *   **作用**：AI 决定是否“唤醒”该技能的唯一路由依据。
*   **第 2 层：指令层（Instruction Layer）** 
    *   **按需加载**：即 `SKILL.md` 正文。
    *   **作用**：规范 AI 的思考路径和输出结构。
*   **第 3 层：资源层（Resource Layer）** 
    *   **动态调用**：包含 `references/`（领域知识）、`scripts/`（验证脚本）等。
    *   **作用**：提供支撑执行的弹药库。

## 三、理论落地：9 大实战案例深度解析

为了让理论不再干瘪，我们将 Datawhale AI 学习中心的 9 个经典实战案例进行拆解，看看这些优秀作品是如何应用 Skill 设计心法的。

### 类别一：教育与认知升级（知识的解构与重组）
*   **案例 1：基于苏格拉底追问+费曼学习法的课程深度学习系统 Skill**
*   **案例 2：数学错题诊断 Skill**
*   **设计心法映射**：**【分阶段工作流】与【强制停顿】**
    *   这两个 Skill 的精髓在于它们**不是直接给答案**。在 `SKILL.md` 的指令层中，它们采用了类似于“教练”的工作流（SOP）。
    *   例如错题诊断，Skill 会强制要求 AI 执行：`阶段1：定位知识点漏洞 -> 阶段2：给出相似提示但不给全解 -> 阶段3：等待用户回复`。这完美体现了 Skill 可以塑造 AI 的“人格与行为边界”。

### 类别二：个人生活管家（数据解析与情绪感知的交织）
*   **案例 3：个人消费账单智能分析 Skill**
*   **案例 4：周末回血局——情绪感知型微旅行助手 Skill**
*   **设计心法映射**：**【资源层依赖】与【情境路由】**
    *   账单分析极度依赖于结构化数据的处理，它必然需要配合 MCP（模型上下文协议）或脚本工具来读取 CSV/Excel，其 `description` 会明确写道：“当用户上传包含金额、日期等消费记录时必须触发”。
    *   微旅行助手的亮点在于“情绪感知”。其元数据层（路由）会捕捉用户诸如“太累了”、“想放松”等情绪触发词，从而加载针对性的疗愈出行知识库（Resource Layer）。

### 类别三：内容创作流水线（从灵感到成品）
*   **案例 5：用 AI 构建自己的内容创作系统 Skill**
*   **案例 6：AI 口播稿 Skill（把松散想法变成能直接开口讲的内容）**
*   **设计心法映射**：**【格式规范】与【风格注入】**
    *   “怎么写，AI 就怎么干”。口播稿 Skill 的指令层必然像一份“编导手册”，强制要求 AI 遵循特定的 Markdown 结构输出（如：`[00:00-00:05] 黄金三秒钩子`），通过规定输出范式，解决 AI 写作通常“干瘪、假大空”的问题。

### 类别四：职场与政务引擎（严谨与规范的代名词）
*   **案例 7：政务公务智能处理 Skill**
*   **案例 8：企业公司调研 Skill**
*   **案例 9：Office 全能处理 Skill**
*   **设计心法映射**：**【内置安全边界】与【API 级精确度】**
    *   涉及政务和企业调研，**安全不是事后补丁，而是内置属性**。这些 Skill 的指令层中通常包含严格的脱敏规则（如 PII 脱敏）和事实核查声明（“所有事实性推测必须标注 [待确认]”）。
    *   对于 Office 处理，Skill 就像一个 API 文档，明确输入参数（文档类型、核心诉求）和输出产物，杜绝废话。

## 四、从零开始：Agent Skill 的标准创建 SOP

结合上述案例的成功经验，如果你想自己动手写一个能真正帮到别人的 Skill，请遵循以下反直觉的顺序：**边界与工作流永远先于文案撰写。**

1.  **痛点挖掘与 I/O 定义**：
    *   不要想“我想让 AI 做什么”，而是问“在这个场景下（如：政务公文流转），输入什么？输出什么格式？绝对不能做什么？”。
2.  **设计分阶段工作流**：
    *   画出决策树。在哪一步需要读取外部资源？在哪一步需要追问用户？
3.  **编写路由（Description）**：
    *   赋予 `description` **攻击性**（Pushy）。写明：“当用户贴出...并附带...时，必须使用此 Skill。触发词包括：xxx”。不留模棱两可的空间。
4.  **撰写指令（SKILL.md）**：
    *   以技术手册或 SOP 的风格编写。使用强制性的结构（如 `## 诊断路径`, `## 必须包含的字段`）。
5.  **配备资源与自动化验证**：
    *   在 `references/` 提供行业标准（如公文格式国标）。编写 `validate_example.py` 等脚本，确保每次 AI 的输出都能被代码验证，而非仅靠肉眼看。

## 五、结语

这 9 个优秀的案例向我们展示了：Agent 开发正在经历一个重要的转折点。**Skill 标志着从 Prompt 工程到软件工程的跃迁。**

当我们将能力封装为独立的、可测试的 `SKILL.md` 单元，AI 能力就具备了**可版本控制、可测试、可协作**的资产特性。学习设计 Skill，本质上是在学习如何拆解人类的高级认知工作流，这项“元技能”将成为未来 AI 时代的底层核心竞争力。
