## 07. `multi-source-evidence-check` —— 多源证据交叉校验 Skill

### 一、 核心架构与底层逻辑
**1. 痛点与绝对边界**
信息来源复杂时容易打架，人眼比对极易漏掉关键细节。
*   **边界约束**：**作为中立的裁判，绝不偏袒某一方，只忠实地高亮冲突点**。

**2. 状态机与工作流设计**
采用辩证逻辑树模型：提取 Source A 核心事实 -> 提取 Source B 核心事实 -> 对比差异 (Conflict) -> 输出置信度评估。

### 二、 实战演示与场景穿透
*   **User输入**：“这是供应商发来的报价说明，这是采购部这边的调研报告，帮我交叉核对一下关于发货周期和违约金条款的描述是否有出入。”
*   **Skill执行过程**：AI 会分左右栏或列表格，把两边咬合不上的条款一条条拽出来，并标红提示。

### 三、 深度心得
这个 Skill 的设计逻辑很像“代码 Diff 工具”。它证明了 AI 在处理繁杂文本时的细致程度可以超越人类，是审计和法务场景的绝佳外脑。
