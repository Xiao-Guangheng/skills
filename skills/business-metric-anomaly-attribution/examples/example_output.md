# 示例输出

- 数据范围与口径判断：2026 年 7 月第 2 周经营周报，关注 GMV、订单量、转化率、曝光量、客单价和退款率的周环比变化
- 核心指标概览：
  - GMV 环比下降 12%
  - 订单量下降 9%
  - 转化率下降 0.8 个百分点
  - 曝光量基本持平
  - 客单价上升 3%
  - 退款率上升 1.5 个百分点
- 异常波动清单：
  - GMV 和订单量同步下滑，属于核心结果异常
  - 转化率下降但曝光量持平，说明问题更偏向转化端而不是流量端
  - 退款率上升，可能进一步拖累有效成交
- 异常分层判断：
  - 结果层：GMV 下滑
  - 转化层：转化率下降是最直接的解释线索
  - 流量层：曝光量持平，暂时不是首要矛盾
  - 活动层：主推活动提前结束，可能影响转化和订单
  - 投放层：预算减少 20%，需要验证是否影响了高质量流量
- 可能原因拆解：
  - 活动提前结束导致转化承接减弱
  - 投放缩减可能使优质流量减少
  - 退款率上升可能影响净成交
  - 客单价上涨未能抵消订单下滑
- 待验证线索：
  - 是否存在渠道结构变化
  - 是否有商品缺货或价格变动
  - 是否存在页面改版、活动切换或竞品促销
  - 是否有品类或 SKU 的局部拖累
- 钻取路径建议：
  - 路径 1（优先级最高）：GMV → 按渠道拆分 → 看各渠道 GMV 和转化率变化 → 判断是哪个渠道拖累 → 如果某个渠道 GMV 明显下滑，进一步查该渠道的投放数据和活动覆盖情况
  - 路径 2：转化率 → 按页面/落地页拆分 → 看各页面转化率变化 → 判断是否是活动页面下架导致 → 如果活动页面对应的流量占比大且转化骤降，可以初步锁定原因
  - 路径 3：退款率 → 按 SKU/品类拆分 → 看高退款 SKU 分布 → 判断是否有集中退款的单品 → 如果是单品问题，需联动运营确认是否涉及质量或描述不符
- 决策权衡框架：
  - 当前两难决策："先恢复投放预算"还是"先修复转化问题"？
  - 方案 A：恢复投放
    - 投入成本：恢复 20% 预算，按原投放金额估算
    - 预期收益：流量回升，如果转化不变则 GMV 按比例回升
    - 风险：如果转化问题未解决，投放效果打折，ROI 可能低于预期
    - 验证周期：1 周可见初步结果
  - 方案 B：先修复转化
    - 投入成本：排查活动页面、SKU 和退款问题，人日约 3-5 天
    - 预期收益：转化回升后，同样流量下 GMV 更高
    - 风险：排查周期内无新增投放，GMV 可能继续承压
    - 验证周期：排查 1 周 + 修复后观察 1 周
  - 推荐优先级：先修转化，再恢复投放；理由是转化问题是当前最直接的拖累项，投放恢复前不解决转化会被放大浪费
- 结论摘要：当前更像"转化与活动承接问题"，不是单纯曝光不足；优先检查活动结束、投放缩减和退款率上升对成交的影响
