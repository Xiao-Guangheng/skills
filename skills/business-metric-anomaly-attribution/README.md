# 业务指标异常归因助手

> 面向「数据智能分析与应用」赛道的参赛 Skill

一个系统化的经营指标异常检测与归因分析 Skill，覆盖"异常检测→分层归因→钻取路径→决策权衡→汇报输出"五阶段决策链路。

---

## 核心能力

### 1. 异常检测与分层归因
- 按结果层、结构层、流量层、转化层、价格层、成本层逐层拆解
- 明确区分"已知事实""合理推测""需要继续验证"三类内容
- 口径和时间范围敏感，不把模糊数据当事实

### 2. 钻取路径建议
- 给出 2-3 条优先级排序的数据钻取路径
- 每条路径包含起点指标、逐层拆分维度和每层想验证的假设
- 标记数据粒度不足的路径，说明需要什么额外数据

### 3. 决策权衡框架
- 当用户面临两难决策（如"加大投放还是优化转化"）时输出结构化权衡
- 每个方案拆解：投入成本、预期收益、风险等级、验证周期
- 推荐优先级并说明依据

### 4. 可汇报输出
- 结论摘要、图表建议、行动项清单可直接用于周报/月报/经营分析会
- 必要时补一版"老板能听懂"的短结论

## 适合什么时候用

| 场景 | 说明 |
| --- | --- |
| 日报/周报/月报复盘 | 自动识别异常指标并生成可汇报的分析 |
| 经营指标异常排查 | GMV/转化/留存/收入/成本波动时快速定位原因 |
| 老板追问的情况 | 输出结构化归因，区分事实和推测 |
| 方案决策辅助 | 两个方案之间做量化对比和风险权衡 |
| 经营分析会准备 | 汇总图表建议、行动项和钻取路径 |

## 使用建议

1. 先提供数据口径、时间范围和业务背景
2. 数据维度不全时先补齐拆分维度再做深度归因
3. 钻取路径按优先级逐条验证
4. 决策权衡框架需人工确认假设前提

⚠️ **AI 生成内容提醒**：本技能包的所有分析和建议均由 AI 生成，请勿据此直接决定投放策略、预算分配或重大业务方向。
