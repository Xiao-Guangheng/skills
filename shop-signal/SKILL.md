# ShopSignal

## 适用场景
你手上有一个商品，想把商品信息、用户评论、客服高频问题和竞品信息整理成能直接用于上新、优化转化和辅助客服的内容包。

## 输入
- 商品名称
- 核心卖点
- 规格参数
- 价格区间
- 用户评论或评价摘要
- 客服高频问题
- 竞品对比信息
- 店铺活动信息

## 标准流程
1. 先识别商品类型、目标人群和最核心的购买动机。
2. 再提炼好评理由、差评风险和用户真实需求。
3. 检查是否缺少关键事实，例如价格、规格、库存、适用场景；缺少时先提示补充，不编造。
4. 生成商品详情页文案，包括标题、卖点分层、场景说明、FAQ 和转化引导。
5. 输出评论洞察摘要，标注高频好评点、差评点、疑问点和改进建议。
6. 如果用户需要客服辅助，再输出可直接使用的回复建议和话术模板。

## 输出格式
- 商品定位判断
- 目标用户
- 核心卖点
- 差评风险
- 用户需求洞察
- 详情页标题候选
- 卖点分层文案
- 场景化描述
- 评论洞察摘要
- 客服回复建议
- FAQ
- 上新/优化建议

## 触发词
- 帮我写详情页
- 分析评论
- 提炼差评点
- 生成卖点文案
- 客服回复建议
- 整理用户需求
- 竞品评论总结
- 商品文案优化
- 详情页改写
- 用户评价分析

## 不应该做什么
- 不编造价格、库存、链接和功效
- 不把未验证的评论当成事实
- 不输出夸大宣传、违规承诺或绝对化表述
- 不把竞品信息写成没有来源的结论
- 不忽略商品类目差异和平台规则

## 验收标准
- 用户能直接拿去写详情页或上新页
- 用户能直接拿去做评论复盘
- 用户能看出差评风险和改进方向
- 用户能把客服回复直接复制使用
- 所有事实性内容都有明确来源或被标记为待补充
